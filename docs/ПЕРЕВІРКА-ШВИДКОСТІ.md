# Перевірка швидкості сайту (робити ПІСЛЯ публікації на Vercel)

Це не можна перевірити до того, як сайт опубліковано з реальною адресою —
інструменти нижче перевіряють живий сайт в інтернеті.

## Крок 1: Google PageSpeed Insights (безкоштовно, найпростіше)

1. Перейдіть на **pagespeed.web.dev**
2. Впишіть адресу сайту, натисніть "Аналізувати"
3. Перевірте окремо **мобільну версію** і **версію для комп'ютера**
   (перемикач угорі) — мобільна зазвичай важливіша, бо більшість
   покупців заходять з телефону

## На що дивитись у результаті

- **Продуктивність (Performance)** — оцінка від 0 до 100. Вище 70 —
  вже добре для інтернет-магазину з фото товарів
- **SEO** — має бути близько 100, якщо сторінки товарів мають Title/
  Description (це вже реалізовано в Етапі 1)
- Якщо оцінка продуктивності низька — найчастіша причина: занадто важкі
  фотографії товарів. Рішення: стискати фото перед завантаженням
  (сервіси на кшталт tinypng.com, безкоштовно) або підключити
  автоматичну оптимізацію зображень Next.js (`<Image>` замість `<img>` —
  це вже окрема невелика доробка коду, за потреби наступний Claude-чат
  може це зробити)

## Крок 2: Перевірка вручну — "чи бачить Google без JS"

1. Відкрийте будь-яку сторінку товару на живому сайті
2. Клацніть правою кнопкою → **"Переглянути код сторінки"** (View Page
   Source — НЕ звичайний "Inspect"/DevTools, а саме "перегляд коду
   сторінки", це принципово інша команда)
3. Знайдіть у відкритому коді назву товару та ціну текстом. Якщо вони
   там є — саме це і означає, що Google бачить сторінку без виконання
   JavaScript. Це головна мета всієї роботи над сайтом.

## Крок 3: Перевірка в самому Google Search Console

Через кілька днів після підключення Search Console (див.
`docs/SEARCH-CONSOLE.md`) — інструмент **"Перевірка URL-адреси"**
(вгорі сторінки, поле пошуку) дозволяє вписати будь-яку адресу вашого
сайту і побачити, як саме її бачить Google, включно зі скріншотом того,
що "бачить" робот Google.
