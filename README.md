# Beauty Parts — Next.js версія сайту

## Що це

Переробка сайту beautyparts.com.ua з одностраничного JS-додатку (SPA) на
Next.js із серверним рендерингом (SSR), щоб Google міг індексувати кожен
товар/категорію/бренд як окрему сторінку — навіть без виконання JavaScript.

## Важливо для Етапу 2: колонка "модель" у Google Таблиці

Сторінки `/model/[slug]` і фільтр за моделлю в каталозі очікують, що у
вашій таблиці товарів є колонка з моделлю машинки, до якої підходить
запчастина (наприклад "Wahl Magic Clip", "Moser ChromStyle"). Назва
колонки за замовчуванням: `model` / `Model` / `compatibleModel` —
див. `lib/sheets.js`, функція `normalizeProduct`. Якщо такої колонки
в таблиці ще немає — додайте її, інакше фільтр і сторінки моделей
просто не будуть заповнюватись даними (сайт при цьому не зламається,
просто розділ буде порожнім).

## Що потрібно ОБОВ'ЯЗКОВО підлаштувати перед запуском

1. **`.env.example` → `.env.local`**: скопіюйте файл, впишіть справжню
   адресу `SHEET_API_URL` (та, що зараз використовується в старому
   `index.html`) і `SITE_URL`.

2. **`lib/sheets.js`, функція `normalizeProduct`**: тут перелічені назви
   полів (`row.name`, `row.price`, `row.brand`...). Якщо колонки у вашій
   Google Таблиці називаються інакше (наприклад, українською або з
   великої літери по-іншому) — підправте цю функцію під реальні назви.
   Найпростіше: відкрийте старий `index.html`, знайдіть, як там читається
   масив товарів (`PRODUCTS` чи відповідь із Sheet API), скопіюйте назви
   полів сюди.

3. **SEO-поля** (`seoTitle`, `seoDescription`, `h1`): якщо хочете, щоб
   адміністратор міг їх редагувати вручну — додайте в Google Таблицю
   три нові колонки з такими назвами. Якщо колонки порожні — Title і
   Description згенеруються автоматично з назви й опису товару.

## Як запустити локально (перевірити перед публікацією)

Потрібен встановлений Node.js (щонайменше 18 версія, з nodejs.org).

```bash
npm install
npm run dev
```

Відкрити у браузері: http://localhost:3000

Перевірити, що Google справді бачить контент без JS: клацнути правою
кнопкою на сторінці товару → "Переглянути код сторінки" (View Page
Source, НЕ звичайний Inspect) — там має бути весь текст товару.

## Як опублікувати (безкоштовно, без свого сервера)

1. Створити акаунт на **vercel.com** (вхід через GitHub)
2. Залити ці файли на GitHub (можна через веб-кнопку "Upload files",
   без командного рядка)
3. У Vercel: "Add New Project" → обрати цей репозиторій
4. У розділі "Environment Variables" вписати `SHEET_API_URL` і `SITE_URL`
   (ті самі значення, що в `.env.local`)
5. "Deploy" — за хвилину отримаєте робочу адресу
6. Прив'язати домен beautyparts.com.ua у налаштуваннях проєкту Vercel

## Структура файлів

```
app/
  layout.js              — загальний каркас сторінки (head, body)
  page.js                — головна сторінка
  sitemap.js             — автогенерація /sitemap.xml
  robots.js              — автогенерація /robots.txt
  product/[slug]/page.js — сторінка одного товару (Title/Description тут)
  category/[slug]/page.js— сторінка категорії
  brand/[slug]/page.js   — сторінка бренду
lib/
  sheets.js              — ВСЯ робота з Google Sheets API тут і тільки тут
```
